# retro

Use after completing a task, feature, or release.

## Output format

1. What went well
2. What was painful or slow
3. Root causes (process, tooling, clarity, coordination)
4. Action items (owner + next trigger)
5. One change to adopt immediately

## Rules

- Keep it constructive and specific.
- Focus on improvements that can be applied next cycle.
- Prefer small, high-leverage process changes.
