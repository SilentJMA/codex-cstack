# plan-product-review

Use when the user describes a feature and we need to confirm we are solving the right problem.

## Goals

- Identify the real user job and desired outcome.
- Distinguish must-have scope from nice-to-have scope.
- Surface hidden assumptions, tradeoffs, and risks early.
- Produce a build-ready problem statement.

## Output format

1. User outcome
2. Proposed scope (v1)
3. 10-star extension (optional next step)
4. Key risks and unknowns
5. Recommendation

## Rules

- Do not jump into code before the recommendation is clear.
- Keep language concrete and decision-oriented.
- Prefer one strong recommendation over many vague options.
